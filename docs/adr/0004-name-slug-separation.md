# ADR-0004 : 커스텀 게시판 생성을 위해 name과 slug를 분리한 이유

## Date

2026.04.28 (화)

## Context

CoreBoard는 고정된 게시판만 제공하는 구조가 아니라, 운영자가 게시판을 직접 생성하고 관리할 수 있는 구조를 목표로 한다.

처음에는 게시판 이름만 있으면 충분하다고 생각했다.

예를 들어 `자유게시판`, `Q&A`, `공지사항` 같은 이름을 게시판 식별값으로 사용할 수도 있다.

하지만 게시판 이름은 사람이 보는 표시값이기 때문에 언제든 변경될 수 있다.

예를 들어 다음과 같은 변경이 발생할 수 있다.

- `Q&A` → `질문답변`
- `자유게시판` → `자유게시판🔥`
- `취업/면접` → `취업·면접 게시판`

이 값을 그대로 URL이나 API 식별자로 사용하면 문제가 생긴다.

예를 들어 `취업/면접`이라는 게시판 이름을 그대로 URL에 사용하면 `/boards/취업/면접/posts`처럼 보이는데, 여기서 `/`는 URL 경로 구분자로 해석될 수 있다.

또한 게시판 이름이 바뀔 때마다 API 경로와 기존 링크도 함께 바뀌는 문제가 생긴다.

따라서 화면에 보여주는 이름과 시스템에서 사용하는 식별자를 분리할 필요가 있었다.

## Decision

```java
private String name;
private String slug;
```

- name (사용자가 화면에서 보는 게시판 이름) : 자유게시판, Q&A ,공지사항, 취업/면접
- slug (URL/API에서 게시판을 식별하는 값) : free, qna, notice, job-interview

```JSON
{
  "name": "취업/면접",
  "slug": "job-interview"
}
```

## Options Considered

### 1. 서버가 slug를 자동 생성한다

```
name = "취업/면접"
slug = "board-7"
```

- 장점
    - 사용자는 게시판 이름만 입력하면 된다.
    - slug 중복이나 형식 오류를 서버가 통제하기 쉽다.
    - URL 안전성이 높다.
- 단점
    - board-7만 보고 게시판 의미를 알 수 없다.
    - 커스텀 게시판이라는 의도가 약해진다.
    - 운영자가 원하는 URL 구조를 직접 정할 수 없다.

### 2. 서버가 slug를 자동 생성한다

```JSON
{
  "name": "취업/면접",
  "slug": "job-interview"
}
```

- 장점
    - 화면 표시 이름과 URL/API 식별자의 역할이 분리된다.
    - 게시판 이름을 바꿔도 slug를 유지하면 기존 URL이 유지된다.
    - URL만 보고 게시판 의미를 파악할 수 있다.
    - 운영자가 커스텀 게시판을 직접 설계한다는 의도가 잘 드러난다.
- 단점
    - slug 중복 검증이 필요하다.
    - slug 형식 검증이 필요하다.
    - 사용자가 잘못된 slug를 입력하지 못하게 제한해야 한다.
    - name과 slug를 모두 입력해야 하므로 생성 화면이 조금 복잡해진다.

## Consequences

- 좋은 점
    - 게시판 이름과 시스템 식별자의 책임이 분리된다.
    - 게시판 이름을 변경해도 slug를 유지하면 기존 링크를 보존할 수 있다.
    - /boards/job-interview/posts처럼 의미 있는 URL을 만들 수 있다.
    - 운영자가 게시판을 직접 구성하는 커스텀 게시판 구조와 잘 맞는다.
- 나쁜 점 (트레이드오프)
    - slug 입력값 검증이 필요하다.
    - 중복 slug를 막아야 한다.
    - 한글, 공백, 특수문자, 슬래시(/) 등을 제한해야 한다.
    - 게시판 생성 API와 관리자 화면에서 name과 slug의 차이를 설명해야 한다.

## Validation Rule

- 소문자 영어 허용
- 숫자 허용
- 하이픈(-) 허용
- 공백 금지
- 한글 금지
- 슬래시(/) 금지
- 중복 금지

| 입력값             | 허용 여부 | 이유          |
|-----------------|------:|-------------|
| `free`          |     O | 안전한 영문 slug |
| `qna`           |     O | 안전한 영문 slug |
| `job-interview` |     O | 하이픈 포함 가능   |
| `취업/면접`         |     X | 한글과 슬래시 포함  |
| `Job Interview` |     X | 대문자와 공백 포함  |
| `job/interview` |     X | 슬래시 포함      |
